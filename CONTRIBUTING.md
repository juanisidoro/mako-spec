# Contributing to MAKO

Thank you for your interest in contributing to the MAKO specification.

## How to Contribute

### Reporting Issues
- Use GitHub Issues for bug reports, feature requests, and spec clarifications
- Provide clear descriptions with examples when possible

### Proposing Changes
1. Fork the repository
2. Create a feature branch (`fix/description` or `feat/description`)
3. Make your changes
4. Submit a Pull Request with a clear description of the change and why it's needed

### Spec Changes
- Changes to `spec.md`, `cef.md`, or `headers.md` require discussion in an Issue before submitting a PR
- Include examples demonstrating the proposed change
- Consider backwards compatibility

### Examples
- New content type examples are welcome
- Follow the existing format in `examples/`
- Include realistic, useful content

## Style Guide

- Use clear, concise language
- Prefer concrete examples over abstract descriptions
- Use RFC 2119 keywords (MUST, SHOULD, MAY) for normative statements in the spec
- Keep markdown formatting consistent with existing files

## Code of Conduct

Be respectful, constructive, and inclusive. We follow the [Contributor Covenant](https://www.contributor-covenant.org/).
